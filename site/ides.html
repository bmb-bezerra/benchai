Liberdade de modelo

Kilo Code, OpenCode, Cline, Aider, OpenHands e Goose são rotas fortes quando o critério principal é alternar modelos, provedores e execução local.

Use quando: custo, privacidade, BYOK, modelos locais ou experimentação forem prioridade.

produtividade diário

Ambiente principal

Claude Code, OpenAI Codex, GitHub Copilot, Cursor e Windsurf cobrem implementação, revisão e tarefas recorrentes em bases reais.

Use quando: o time quer menos configuração e mais fluidez no dia a dia.

empresa governança

SDLC governado

GitHub Copilot, Amazon Q, Tabnine, Sourcegraph, Qodo, Harness.io e OpenHands self-hosted fazem mais sentido quando compliance e delivery importam.

Use quando: há exigência de auditoria, privacidade, PR review, CI/CD ou políticas corporativas.

Abertos e multi-modelo

Priorize aqui quando a decisão principal for liberdade de provedor, BYOK, execução local, self-hosting ou troca rápida de modelos.

liberdade operacional
multi-modelo open source

Kilo Code

Priorize quando a métrica principal for liberdade de modelo, BYOK, modelos locais e execução em VS Code, JetBrains, CLI ou cloud.

Atenção: claim de 500+ modelos vem do fornecedor; valide custo, latência, permissões e qualidade por tarefa.

open source terminal/IDE

OpenCode

Boa opção quando o time quer agente aberto, sem ficar preso a um provedor, com possibilidade de usar Claude, GPT, Gemini e outros modelos.

Use para: tarefas de terminal, edição multi-arquivo, fluxos locais e experimentação com provedores diferentes.

VS Code BYOK/local

Cline

Escolha quando quiser controle fino no editor, aprovação por etapa, Plan/Act, MCP, modelos locais e endpoints OpenAI-compatible.

Atenção: autonomia alta exige revisão de diffs, limites de comando e testes locais antes do merge.

terminal git-native

Aider

Excelente para quem vive no terminal e quer edição multi-arquivo com commits, diffs e integração natural com Git.

Use para: refactors controlados, correções pontuais, geração com testes e workflows em repositórios existentes.

orquestração self-host

OpenHands

Use quando a necessidade for rodar agentes em backends locais, remotos ou cloud, com controle maior sobre execução e automações.

Melhor encaixe: times avançados que querem plataforma de agentes, não apenas chat no editor.

desktop/CLI MCP/ACP

Goose

Boa rota para automações locais que combinam desktop, CLI, API, provedores diferentes e extensões MCP.

Use para: tarefas além de código, automações de engenharia, pesquisa, análise e workflows reaproveitáveis.

editor rápido ACP

Zed + ACP

Inclua como editor/harness quando o objetivo for usar vários agentes externos, como Claude, Codex, OpenCode e Copilot, dentro de uma superfície rápida de revisão.

Atenção: agentes externos mantêm autenticação, billing e configuração próprios; Zed é a superfície de edição e revisão.

terminal Qwen/open source

Qwen Code

Considere quando o time quer um agente open source de terminal otimizado para a família Qwen e para entendimento de codebases grandes.

Atenção: apesar de aberto, o melhor encaixe tende a ser com modelos Qwen; compare contra Cline, OpenCode e Aider se a prioridade for provedor livre.

terminal multi-modelo

Crush

Alternativa de TUI/CLI para quem quer uma experiência de terminal polida com ferramentas, código e workflows conectados ao LLM escolhido.

Use para: devs que preferem terminal e querem trocar modelos sem trocar a superfície de trabalho.

Produtividade diária

Ferramentas para implementação, revisão, debug e manutenção frequente em codebases reais.

fluxo principal
GitHub-first IDE + PR

GitHub Copilot

Melhor default quando o time já vive em GitHub e precisa de adoção ampla, agent mode, PRs, issues e governança centralizada.

Use para: autocomplete, chat, agent mode local, cloud agent, PR review e integração direta com GitHub.

tarefa longa review/refactor

Claude Code

Forte para refatorações, debugging, leitura de codebase, revisão crítica e execução multi-arquivo quando julgamento pesa mais que latência.

Atenção: use checkpoints, testes e revisão humana antes de aceitar mudanças grandes.

OpenAI cloud/CLI/IDE

OpenAI Codex

Use quando quiser um agente OpenAI para planejar, escrever, revisar e entregar código em fluxos locais ou cloud.

Atenção: substitui a leitura estreita de "Codex CLI"; o produto cobre CLI, editor, app e cloud tasks.

AI IDE execução diária

Cursor

Bom default para implementação multi-arquivo, navegação de codebase, agente no editor, regras, MCP e fluxo diário de desenvolvimento.

Atenção: use checkpoints, diffs pequenos, regras de repositório e validação local para conter regressões.

agentic IDE Cascade

Windsurf

Alternativa ao Cursor com foco em Cascade, troca de modelos, modelos SWE próprios, Claude, GPT e opções BYOK.

Atenção: separar de Devin; Windsurf é IDE/agentic editor, Devin é cloud/software engineer autônomo.

Google multi-agente

Google Antigravity / Antigravity CLI

Preferir este card ao antigo Gemini Code Assist quando o foco for plataforma agentic do Google, com editor, terminal, browser, artifacts e orquestração.

Atenção: Gemini CLI/Gemini Code Assist devem ser tratados como rota em transição; verificar disponibilidade por plano antes de recomendar.

JetBrains adoção

JetBrains Junie

Relevante quando o time usa IntelliJ, PyCharm, WebStorm ou outras IDEs JetBrains e quer agente integrado sem trocar ambiente.

Use para: execução em IDE, terminal ou CI/CD; validar suporte BYOK, MCP e ACP conforme plano/versão.

spec-driven agentic IDE

Kiro

Boa opção quando o problema é transformar prompts em requisitos, design técnico, tarefas sequenciadas e execução por agentes paralelos.

Use quando: governança de requisitos e rastreabilidade forem tão importantes quanto gerar código rápido.

AWS modernização

Amazon Q Developer

Faz sentido quando a restrição principal é operar, modernizar, revisar ou proteger software dentro do ecossistema AWS.

Use para: CLI, IDE, testes unitários, revisão de segurança, modernização Java e integração com serviços AWS.

Enterprise, review e governança

Rotas para codebase grande, privacidade, PR review, automação de delivery e controle corporativo.

SDLC governado
codebase grande enterprise

Sourcegraph Amp / Cody

Indicado para organizações que precisam de contexto de codebase em escala, busca, entendimento de repositórios e agentes com modelos fortes.

Use quando: o gargalo for contexto em monorepo, multi-repo ou código legado, não apenas autocomplete.

privacidade enterprise

Tabnine

Escolha por restrição de dados quando VPC, on-prem, air gap, compliance ou seleção controlada de modelos forem obrigatórios.

Atenção: compliance e isolamento de dados vêm antes de benchmark genérico.

PR review qualidade

Qodo

Use como camada de qualidade para revisar PRs, encontrar bugs, checar cobertura de testes, documentação, padrões e riscos de segurança.

Atenção: complementa IDEs/agentes de geração; não substitui revisão de arquitetura e ownership do time.

cloud agent autonomia

Devin

Indicado para tarefas delegáveis em cloud, tickets, bugs, features, ferramentas internas e manutenção, sempre com revisão humana de resultado.

Atenção: autonomia longa exige escopo claro, custo monitorado, limites de permissão e critérios de aceite.

DevOps pipeline-native

Harness.io Worker Agents

Use quando o objetivo for colocar agentes dentro de pipelines CI/CD, segurança, IaCM, SCS, STO ou estágios customizados.

Encaixe: é o Harness mais literal: menos IDE, mais automação governada dentro do ciclo de delivery.

enterprise organização

Augment Code / Cosmos

Inclua como opção enterprise quando a dor principal for coordenar agentes em escala organizacional e reduzir perda de contexto em codebases grandes.

Atenção: tratar como plataforma enterprise; validar integrações, privacidade e custo diretamente com o fornecedor.

App builders e prototipagem

Use estes quando a entrega principal é transformar uma ideia em app, demo ou protótipo funcional com pouco setup.

prototipagem
app builder prototipagem

Replit Agent

Escolha para transformar ideia em app ou site funcional com ambiente, código, preview e deploy no mesmo lugar.

Use para: MVPs, protótipos e apps pequenos/médios; hardening e arquitetura ainda precisam de revisão técnica.

browser web apps

Bolt.new

Bom para protótipos web rápidos no navegador, com prompt, execução, edição e deploy sem setup local.

Use para: demos, landing pages e apps full-stack simples; valide dependências, segurança e deploy final.

Vercel full-stack web

v0

Forte para UI, protótipos de produto, apps web full-stack e handoff para times que usam ecossistema Vercel/React.

Use para: explorar produto e front-end; revisar código, acessibilidade, estado, auth e backend antes de produção.

vibe coding app builder

Lovable

Boa opção para founders, PMs e times que querem partir de descrição em linguagem natural para protótipos e apps web funcionais.

Atenção: revisar segurança, privacidade, permissões, dados e qualidade do código antes de escalar.

no-code business app

Base44

Indicado para apps internos e protótipos de negócio quando a pessoa quer criar aplicação funcional com palavras, sem setup técnico.

Atenção: bom para velocidade inicial; avaliar portabilidade, exportação, auth, dados e lock-in.

Como ler os cards

"500+ modelos", "15+ provedores", "70+ extensões" e claims similares indicam posicionamento público do fornecedor. Valide custo, latência, permissões e qualidade no repositório real.

Benchmarks: mantenha comparações numéricas em benchmarks.html para não misturar superfície operacional com ranking de modelo.
